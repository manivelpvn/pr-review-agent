Description

Let’s add a new agent to the workflow. We’ve already used various tools to retrieve context from our GitHub repo. Now, we need to add another agent that uses this information to generate the final comment. Once the report is generated, we will store it in the state for the reviewer agent to review and finally post it on GitHub.

Since we will now have two agents, we need to maintain state so that the agents can communicate effectively. Additionally, we need to use AgentWorkflow to orchestrate the two agents. But first, let’s create tools to manage state for our multi-agent setup.

So far, we need to maintain the following in state:

  • context — the context gathered by the ContextAgent;

  • draft_comment — the first comment generated by the CommentorAgent.

Once the context agent gathers information and generates a summary, it should use another tool to add that summary to the state. Additionally, create another function that, given the draft_comment, saves it in the current state:

current_state = await ctx.get("state")
current_state["draft_comment"] = draft_comment
await ctx.set("state", current_state)

Once you have these functions, add them to the tools list for the corresponding agents. That’s it, the agents can now access the state and also use it to pass information around. Now, let’s work on the CommentorAgent.

  • CommentorAgent

Create a new FunctionAgent — the commenter agent — that uses the retrieved contexts to create a PR comment. Via the system prompt, we give it all the rules and guidelines needed to create a PR comment as a human reviewer would. You should use the following system prompt for this agent:

You are the commentor agent that writes review comments for pull requests as a human reviewer would. \n 
Ensure to do the following for a thorough review: 
 - Request for the PR details, changed files, and any other repo files you may need from the ContextAgent. 
 - Once you have asked for all the needed information, write a good ~200-300 word review in markdown format detailing: \n
    - What is good about the PR? \n
    - Did the author follow ALL contribution rules? What is missing? \n
    - Are there tests for new functionality? If there are new models, are there migrations for them? - use the diff to determine this. \n
    - Are new endpoints documented? - use the diff to determine this. \n 
    - Which lines could be improved upon? Quote these lines and offer suggestions the author could implement. \n
 - If you need any additional details, you must hand off to the Commentor Agent. \n
 - You should directly address the author. So your comments should sound like: \n
 "Thanks for fixing this. I think all places where we call quote should be fixed. Can you roll this fix out everywhere?"

  • workflow

The final piece is the workflow orchestrator. This will handle interactions and handoffs between the agents. To create this, you need to provide it with the agents we created, the first agent to call, and also set the initial state as follows:

workflow_agent = AgentWorkflow(
    agents=[context_agent, commentor_agent],
    root_agent=commentor_agent.name,
    initial_state={
        "gathered_contexts": "",
        "draft_comment": ""
    },
)

Note that the first agent we are calling is the commenter agent.

Finally, run the workflow, and stream the outputs as before:

from llama_index.core.agent.workflow import AgentOutput, ToolCall, ToolCallResult
import asyncio

async def main():
    query = input().strip()
    prompt = RichPromptTemplate(query)

    handler = workflow_agent.run(prompt.format())

    current_agent = None
    async for event in handler.stream_events():
        if hasattr(event, "current_agent_name") and event.current_agent_name != current_agent:
            current_agent = event.current_agent_name
            print(f"Current agent: {current_agent}")
        elif isinstance(event, AgentOutput):
            if event.response.content:
                print("\\n\\nFinal response:", event.response.content)
            if event.tool_calls:
                print("Selected tools: ", [call.tool_name for call in event.tool_calls])
        elif isinstance(event, ToolCallResult):
            print(f"Output from tool: {event.tool_output}")
        elif isinstance(event, ToolCall):
            print(f"Calling selected tool: {event.tool_name}, with arguments: {event.tool_kwargs}")


if __name__ == "__main__":
